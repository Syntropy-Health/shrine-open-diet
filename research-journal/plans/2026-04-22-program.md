# Diet-OS Program Plan

> **For agentic workers:** This is a program-level roadmap of seven subsystems. Each subsystem gets its own detailed TDD plan under `research-journal/plans/`. Execute subsystems in dependency order; within a subsystem, execute tasks in order.

**Goal:** Ship two peer-reviewed publications (primary β + companion γ) anchored on a provenance-grounded agentic harness over a unified diet-bioactivity-TCM knowledge graph.

**Focus pivot (2026-04-22):** Frame the work as a **clinical research team** application — a multi-agent system that supports clinician-researcher panels evaluating dietary/herbal interventions with grounded evidence, not a consumer dietary advisor. Subsystem H (AG2-implemented research team) is the **primary implementation** and case study; standalone Subsystem B (intake agent) is deferred — its schema lives inside H as Stage 1.

**Architecture:** Subsystems A → H → F → G are the critical path. Subsystems B, C, D, E are absorbed into H (their components ship as modules inside the H tree, see `2026-04-22-subsystem-h-clinical-research-team.md`).

**Tech Stack:** Python 3.10+, LightRAG (HKUDS), Neo4j Aura, better-sqlite3, pytest, Bayesian Optimization (scikit-optimize or Ax), OpenAI / Anthropic SDK.

## The seven subsystems

### Subsystem B — Stage 1 Clinical Intake Agent  *(DEFERRED — absorbed into H)*

**Status:** **Deferred per directive 2026-04-22.** B's clinical-intake schema (OPQRST + SOCRATES + NCP/ADIME) is now embodied as **Stage 1 of Subsystem H** rather than a standalone service. A standalone B may be revisited if a production deployment needs intake decoupled from the panel. The schema, prompts, and validation logic still live in `agents/intake_schema.py` (created during H) and could be extracted later.
**Original scope (preserved for reference):** Python service that transforms raw user query into `structured_intake` JSON conforming to OPQRST + SOCRATES + NCP/ADIME schema; clarification loop bounded at 2 rounds; red-flag classifier.

### Subsystem C — Stage 2 KG retrieval + typed-chain extractor  *(absorbed into H as a shared tool)*

**Status:** Now implemented as the `kg_query` AG2 tool inside Subsystem H — exposed to every panel role.
**Produces:** Service that maps `structured_intake` → LightRAG dual-level keywords → hybrid-mode query → typed-chain extractor → candidate_chains[] with source_ids, weights, evidence-tier labels.
**Key outputs (within H):**
- `shrine-diet-bioactivity/agents/tools/kg_query.py` (AG2 `register_for_execution` wrapper)
- `shrine-diet-bioactivity/agents/tools/chain_extractor.py`
- Tests under `agents/tests/test_kg_query.py`

### Subsystem D — Stage 3 Adaptive expert panel  *(absorbed into H)*

**Status:** Now implemented as the AG2 `GroupChat` + 6 `ConversableAgent` roles inside Subsystem H.
**Produces:** MDAgents-style triage + 6-role agents with MedAgents-style debate-consensus; dissenting-minority report generator.
**Key outputs (within H):**
- `shrine-diet-bioactivity/agents/panel/triage.py`
- `shrine-diet-bioactivity/agents/panel/roles/{dietitian,pharmacologist,tcm,research_scientist,safety,defer}.py`
- `shrine-diet-bioactivity/agents/panel/debate.py`

### Subsystem E — Stage 4 Calibrator + provenance formatter  *(absorbed into H)*

**Status:** Now implemented as the final-stage AG2 agent inside Subsystem H.
**Produces:** Bayesian linear fusion with BayesOpt-tuned weights, Platt/isotonic calibration, Dempster-Shafer ablation baseline, provenance-chain JSON formatter with Cypher faithfulness verification.
**Key outputs (within H):**

- **Coverage:** ≥ 80% per subsystem's test module.
- **Pinning:** LLM model versions pinned + date-stamped per eval run; random seeds pinned for splits.
- **Review:** `code-reviewer` agent after each subsystem's implementation is complete; `pr-test-analyzer` before marking evaluation complete.
- **Plans written iteratively:** B–G plans get written as A nears completion.
